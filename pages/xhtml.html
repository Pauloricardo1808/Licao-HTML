O que é XHTML?

XHTML significa Extensible Hyper Text Markup Language.

XHTML é uma versão de HTML mais rígida e baseada em XML.

XHTML é HTML definido como um aplicativo XML.

XHTML é suportado por todos os principais navegadores.

Por que XHTML?

XML é uma linguagem de marcação onde todos os documentos devem ser marcados corretamente (ser "bem formados").

O XHTML foi desenvolvido para tornar o HTML mais extensível e flexível para trabalhar com outros formatos de dados (como XML). Além disso, os navegadores ignoram erros nas páginas HTML e tentam exibir o site mesmo que apresente alguns erros de marcação. Portanto, o XHTML vem com um tratamento de erros muito mais rígido.

Se você deseja estudar XML, leia nosso Tutorial de XML.

As diferenças mais importantes do HTML

  • <!DOCTYPE> é obrigatório
  • O atributo xmlns em <html> é obrigatório
  • <html>, <head>, <title> e <body> são obrigatórios
  • Os elementos devem sempre estar adequadamente aninhados
  • Os elementos devem estar sempre fechados
  • Os elementos devem estar sempre em letras minúsculas
  • Os nomes dos atributos devem estar sempre em letras minúsculas
  • Os valores dos atributos devem sempre ser citados
  • A minimização de atributos é proibida

XHTML - <!DOCTYPE ....> é obrigatório

Um documento XHTML deve ter uma declaração XHTML <!DOCTYPE>.

Os elementos <html>, <head>, <title> e <body> também devem estar presentes, e o atributo xmlns em <html> deve especificar o namespace xml do documento.

Exemplo

Aqui está um documento XHTML com um mínimo de tags obrigatórias:

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN"
    http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
    <html xmlns="http://www.w3.org/1999/xhtml">
    <head>
    <title>Title of document</title>
    </head>
    <body>

    
some content here...


    </body>
    </html>

Elementos XHTML devem ser aninhados corretamente

Em XHTML, os elementos devem sempre estar adequadamente aninhados uns nos outros, assim:

Correto: <b><i>Some text</i></b>

Errado: <b><i>Some text</b></i>

Elementos XHTML devem sempre ser fechados

Em XHTML, os elementos devem estar sempre fechados, assim:

Correto: <p>This is a paragraph</p>

Errado: <p>This is a paragraph<p>This is another paragraph</p>

Elementos vazios XHTML devem sempre ser fechados

Em XHTML, os elementos vazios devem sempre ser fechados, assim:

Correto: A break: <br /> | A horizontal rule: <hr /> | An image: <img src="happy.gif" alt="Happy face" />

Errado: A break: <br> | A horizontal rule: <hr> | An image: <img src="happy.gif" alt="Happy face">

Os elementos XHTML devem estar em letras minúsculas

Em XHTML, os nomes dos elementos devem estar sempre em letras minúsculas, assim:

Correto: <body> | <p>This is a paragraph</p>

Errado: <BODY> |<P>This is a paragraph</P>

Os nomes dos atributos XHTML devem estar em letras minúsculas

Em XHTML, os nomes dos atributos devem estar sempre em letras minúsculas, assim:

Correto: <a href="https://www.youtube.com">Visite o Youtube</a>

Errado: <a HREF="https://www.youtube.com">Visite o Youtube</a>

Valores de atributos XHTML devem ser citados

Em XHTML, os valores dos atributos devem sempre ser citados, assim:

Correto: <a href="https://www.youtube.com">Visite o Youtube</a>

Errado: <a href=https://www.youtube.com> Visite o Youtube</a>

A minimização de atributos XHTML é proibida

Em XHTML, a minimização de atributos é proibida, assim:

Correto: <input type="checkbox" name="vehicle" value="car" checked="checked" />

Errado: <input type="checkbox" name="vehicle" value="car" checked />
